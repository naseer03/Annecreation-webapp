üîê Authentication

Most product operations require authentication. Here's how to authenticate:

Customer Authentication

# Login as customer to get access token curl -X POST http://localhost:5000/api/customers/login \ -H "Content-Type: application/json" \ -d '{ "email": "customer@example.com", "password": "password123" }' # Response will include accessToken # Use it in subsequent requests: # -H "Authorization: Bearer YOUR_ACCESS_TOKEN"

Admin Authentication

# Login as admin to get access token curl -X POST http://localhost:5000/api/admin/login \ -H "Content-Type: application/json" \ -d '{ "username": "admin", "password": "admin123" }'

üåê Public Routes (No Authentication Required)

GETGet All Products

Endpoint: /api/products

Retrieves a paginated list of products with filtering and sorting options.

‚úÖ Public Access - No authentication required
Query Parameters:
  • page - Page number (default: 1)
  • limit - Items per page (default: 10, max: 100)
  • category - Filter by category ID
  • manufacturer - Filter by manufacturer ID
  • search - Search in name, model, description
  • price_min - Minimum price filter
  • price_max - Maximum price filter
  • sort - Sort by: name_asc, name_desc, price_asc, price_desc, date_added
  • status - Filter by status (true/false)
Examples:
# Basic request curl -X GET "http://localhost:5000/api/products" # With pagination curl -X GET "http://localhost:5000/api/products?page=2&limit=20" # Search products curl -X GET "http://localhost:5000/api/products?search=laptop&sort=price_asc" # Filter by category and price range curl -X GET "http://localhost:5000/api/products?category=5&price_min=100&price_max=500" # Filter by manufacturer curl -X GET "http://localhost:5000/api/products?manufacturer=3&sort=name_asc"

GETGet Product by ID

Endpoint: /api/products/{id}

Retrieves detailed information about a specific product. Increments view count.

‚úÖ Public Access - Authentication optional (enhanced data for authenticated users)
# Get product details curl -X GET "http://localhost:5000/api/products/123" # With authentication (shows file upload details) curl -X GET "http://localhost:5000/api/products/123" \ -H "Authorization: Bearer YOUR_ACCESS_TOKEN"

GETGet Product Images

Endpoint: /api/products/{id}/images

Retrieves all images associated with a product (main + additional images).

‚úÖ Public Access - No authentication required
# Get all product images curl -X GET "http://localhost:5000/api/products/123/images"

GETServe Product Images

Endpoint: /image/{filepath}

Serves product images from the catalog directory with caching headers.

‚úÖ Public Access - Rate limited to 100 requests per 15 minutes
# Get product image curl -X GET "http://localhost:5000/image/catalog/product/laptop-123.jpg" # Get category image curl -X GET "http://localhost:5000/image/catalog/category/electronics.png"

‚ö†Ô∏è Error Handling

The API returns standardized error responses:

Common HTTP Status Codes:

  • 200 - Success
  • 201 - Created
  • 400 - Bad Request (validation errors)
  • 401 - Unauthorized (missing/invalid token)
  • 403 - Forbidden (insufficient permissions)
  • 404 - Not Found
  • 409 - Conflict (duplicate data)
  • 429 - Too Many Requests (rate limited)
  • 500 - Internal Server Error

Error Response Format:

{ "message": "Error description", "error": "Detailed error message", "status": "error", "type": "validation", "errors": [ { "field": "price", "message": "Price must be a positive number", "value": -10 } ] }

File Upload Errors:

# File too large { "message": "File too large", "error": "Maximum file size is 5MB for images, 50MB for other files" } # Invalid file type { "message": "Invalid file type. Allowed: JPEG, PNG, GIF, WebP" } # Missing file { "message": "No image file provided" }

Authentication Errors:

# Missing token { "message": "Token missing" } # Invalid token { "message": "Invalid or expired token" } # Insufficient permissions { "message": "Admin access required" }
